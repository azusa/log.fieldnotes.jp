# Seasarプロジェクトのプロダクトはバージョン番号の付け方についてポリシーが統一できてない

というのはよく言われることなのですが....[\*1](#f1 "とはいえ、プロジェクトの性格上、それが難しいこともご了承いただきたく。")

S2Dao-CodeGenもちょっと今回はやらかした感がありますね、RCに機能追加しちゃってる^^;betaが適当でしたね...

ともあれ、S2Dao-CodeGenのバージョニングについては、以下のように考えています。

- マイナーバージョン(0.4.\*の「4」)が同じ間については、生成したソースならびに設定ファイル(pom.xml,CodeGen.dicon)の互換性を保ちます。
- リビジョンアップ(0.4.1の「1」)時に、生成したソースならびに設定ファイル(pom.xml,CodeGen.dicon)に機能追加を行う場合があります。
- マイナーバージョンのリリース前のRC、betaについては、互換性を維持しない変更を加える場合があります。

以上、よろしくお願いします。

[\*1](#fn1):とはいえ、プロジェクトの性格上、それが難しいこともご了承いただきたく。
