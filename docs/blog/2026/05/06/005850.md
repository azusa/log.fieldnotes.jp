# 脆弱性情報に関する※NISTの方針転換の方針に関する社内へのSlack案内テンプレ

※ AI生成です

```
*【セキュリティ情報】NVDの脆弱性データ方針転換について* 

*■ まず前提：CVSSスコアとは何か*
ソフトウェアに脆弱性が発見されると、世界共通の識別番号「CVE」が付与されます。ニュースで「〇〇に致命的な脆弱性」と報道される「致命的」の根拠は、CVSSスコア（0〜10点）の区分に基づいています。
```

```
0.0        → None
0.1 ～ 3.9 → Low
4.0 ～ 6.9 → Medium
7.0 ～ 8.9 → High
9.0 ～10.0 → Critical（致命的）
```

```
このスコアは「ネットワーク越しに攻撃できるか」「認証なしで悪用できるか」「機密性・完全性・可用性への影響はどの程度か」などのメトリクスを組み合わせて算出されます。

*CVE番号だけでは深刻度・影響製品は分かりません。* これに CVSS スコアや影響製品リスト（CPE）を付加する「エンリッチメント」作業を一手に担ってきたのが、NISTが運営する NVD（National Vulnerability Database）です。Dependabot などのスキャナーも、この NVD のデータを前提に動作しています。

─────────────────────────
*■ 今回の変更：NVD エンリッチメントの大幅縮小（2026年4月15日施行）*

CVE 申請件数が 2020〜2025 年で *263% 増加* し、2025 年だけで約 49,000 件が登録されました。NIST は過去最多となる 42,000 件をエンリッチしましたが、増加ペースに追いつけず、方針を転換しました。

今後エンリッチメントの対象となるのは以下のみです。

• *CISA KEV カタログ掲載 CVE*（実際の悪用が確認済み） → 受領後1営業日以内
• 米連邦政府が使用するソフトウェアに影響する CVE
• 大統領令 14028 が定めるクリティカルソフトウェアに影響する CVE

上記以外の CVE は `Not Scheduled` ステータスとなり、CVSS スコアや影響製品情報が自動では付与されなくなります。全 CVE の *約 80%* が対象外となる見込みです。

─────────────────────────
*■ 私たちの業務への影響*

:warning: *Dependabot・SCA スキャナーの検出漏れリスク*
CPE（製品識別子）が付与されない CVE はスキャナーが検出できません。「アラートが出ていないから安全」が成立しなくなります。

:warning: *CVSS スコア閾値による機械的フィルタの誤判断リスク*
CNA（ベンダー等）のスコアと NIST スコアは半数超で乖離があります。「スコア7未満はスキップ」などのルールの機械適用は見直しが必要です。

:warning: *顧客向けセキュリティ報告書への影響*
NVD スコアを一次情報として引用している場合、スコア未付与の CVE が報告から漏れる可能性があります。

─────────────────────────
*■ 対応のお願い*

1. *CISA KEV カタログ* を脆弱性トリアージの一次ソースとして参照してください
   → https://www.cisa.gov/known-exploited-vulnerabilities-catalog
2. CVSSスコアのみによる機械的なフィルタリングを廃止または補完し、*EPSSスコア*（悪用確率）との併用を検討してください
3. 利用 OSS のベンダーアドバイザリ（npm security advisories 等）を Dependabot に依存せず直接購読してください
4. 顧客向けの脆弱性管理規程・報告テンプレートに `Not Scheduled = 低リスク` と読める表現がないか確認してください

:information_source: *「Not Scheduled」は NIST のリソース制約を示すものであり、その CVE が低リスクであることを意味しません。*

詳細は NIST 公式アナウンス（2026年4月15日）を参照してください。
```
